# EXHIBIT A  PROJECT SPECIFICATION 

Document titled "OHW Weight Management App  V1 Project Specification" dated May 1st, 2025 is incorporated by reference. 

## OHW Weight Management App - V1 Project Specification - 5/01/25 

### Complete Feature List 

#### 1. User Authentication & Onboarding 

- Secure patient account creation and login with email/password 
- Provider/clinic account creation and login with role-based permissions 
- HIPAA-compliant authentication with session management and timeout features 
- Comprehensive initial onboarding questionnaire based on the Weight Loss Intake Form, including:
  - Patient demographics (name, age, gender, contact information, ethnicity) 
  - Detailed health history (past health events, current medications, supplements) 
  - Comprehensive comorbidity assessment for insurance prior-authorization 
  - Family medical history for risk factor identification 
  - Contraindication screening for medications (gastroparesis, pancreatitis, thyroid cancer) 
- Goals setting in SMART goal format with: 
  - Weight targets with achievable timelines 
  - Lifestyle change options 
  - Energy/mobility improvement goals 
  - Visual roadmap to provide targets and metrics 
- Short (5-10 min) module showing the features of the app 
- Additionally so they can get used to the rewards/gamification system that we use in the app 
- CDC-compliant demographics collection for program recognition requirements 

#### 2. Food Tracking 

- Manual food entry system with intuitive dropdown menus and button selections for ease of use 
- Visual portion size representation using cup, tablespoon, teaspoon equivalents 
- Meal categorization system (breakfast, lunch, dinner, snacks) with timestamps 
- "Favorite meals" functionality to simplify repeated logging 
- Basic nutrition calculation showing calories, macronutrients (carbs, protein, fats) 
- Optional photo logging of meals as backup tracking method 
- Detailed beverage/liquid tracking with categorization (water, coffee/tea, alcohol, etc.) 
- Cooking oils/seasonings logging to capture hidden calories 
- Standardized portion guide with visual references 
- Specific tracking for: 
  - Daily liquid consumption with targets 
  - Snack and main meal differentiation 
  - MyPlate visualization for balanced eating guidance 
- Ability to log restaurant/chain food items with simplified selections (with 3rd party API) 

#### 3. Fitness Tracking 

- Weekly goal tracking toward CDC-required 150 minutes of physical activity 
- Activity type categorization (cardio/weights) with simple dropdown selection 
- Basic movement logging for common activities (walking, biking, running, elliptical) 
- Three-level intensity tracking system (light, moderate, vigorous) 
- Weekly progress visualization with motivational elements 
- Activity categories designed for patients with mobility limitations 
- Visual weekly cadence display showing completed vs. remaining minutes 
- Simple data entry designed for patients unfamiliar with fitness tracking 
- Goal-setting based on program length and weight loss targets 

#### 4. Weight & Measurements 

- Manual weight entry with option for metric or imperial units 
- Then, photo functionality so patients can take a photo inside the app (no upload function) that complies with CDC guidelines (patient's feet on scale showing the weight) 
- Waist circumference tracking for visceral fat assessment 
- Automated BMI calculation with category indication 
- Before/after comparison visualization 
- Detailed progress charts showing trends over time 
- BMI target calculation based on healthy ranges 
- Weight loss target recommendation with realistic timeframes 
- Timeline visualization showing projected progress 
- Personalized nutrition recommendations (Kcals/day) based on goals 
- Fitness minutes recommendations calibrated to weight loss goals and current weight & activity levels (algorithm) - using a system that Anna has 

#### 5. Patient Dashboard 

- Clean, intuitive weight loss progress visualization with trend lines 
- Weekly and cumulative fitness minutes tracking with goal comparison 
- Nutrition summary showing meal balance and caloric distribution 
- Daily and weekly goals overview with completion indicators 
- Visual roadmap showing progress toward targets with milestone markers 
- Simple gamification elements (points/badges) tied to consistent tracking 
- Summary metrics designed for 5-10 minute review 
- Weekly progress indicators for at-a-glance assessment 
- Visual representations of goals vs. actual performance 
- Celebration animations for achieved milestones 
- Risk Reduction modeling - basic modeling incorporating fitness amount/ fitness minutes and weight loss, based on examples sent to us. Focus on teaching risk reduction for this V1, more advanced risk stratification for V2. 
- Labs section where patients can view their uploaded lab PDFs 

#### 6. Provider Dashboard 

- Comprehensive patient summary view designed for 5-10 minute review sessions 
- Weight tracking with detailed graph visualization showing program progression 
- Exercise completion rates with weekly and monthly trend visualization 
- Food journal review interface with photo gallery option 
- Critical metrics visualization highlighting key health indicators 
- One-click snapshot export functionality (JPEG/PDF) for EHR integration 
- Standardized data export for CDC DPP reporting (Excel format) 
- Patient comparison to baseline metrics at program start 
- Patient list with sorting and filtering capabilities 
- Maximum 5-10 minutes review time per patient design philosophy 
- Exportable snapshots for integration with PatientNow and other EHR systems 
- Email reminder for the provider every 6 months to do labs for patients & in the dashboard as well 
- ICD-10 codes to prescribe/qualify patient for weight loss medications 
  - For V1, a very simple version - the ICD10 codes are very simple, this feature can just list all of the possible ones based on the intake form, keeping it very simple for V1. V2 can have a feature that uses all the lab data & has an AI feature to help write up the case according to the ICD-10 code for why they're requesting that prescription to get insurance to pay for it 
- Titration schedule for patients based on individual data - Anna has an algorithm/ way of doing it, we can implement this for the V1. 

#### 7. Educational Content 

- Microlearning modules based on CDC DPP curriculum delivered in digestible segments 
- Configurable delivery cadence options (1x or 3x weekly) based on user preference 
- Basic portion size guidance with visual aids 
- Simple knowledge checks to verify understanding after modules 
- Nutrition fundamentals education covering: 
  - Portion control principles 
  - Calorie/macro tracking basics 
  - Balanced plate visualization 
  - Visual aids for measurements 
- Optional learning tasks for extra engagement points 
- Basic CDC-compliant educational content preformatted to guidelines 
- Knowledge verification system to confirm content review 
- Detailed and comprehensive FAQ section in place of an AI Chatbot (chatbot will come in V2, not V1) 

#### 8. Patient Engagement 

- Weekly follow-up questionnaire designed to identify challenges and track progress 
- Configurable notification system with customizable alert preferences 
- Calendar integration (.ics) for reminders of appointments and tasks 
- Structured goal re-evaluation using SMART goal format 
- Point/reward system tied to consistent tracking and goal achievement 
- Pre-visit questionnaire delivered 24-48 hours before appointments 
- Comprehensive side effects tracking for medication management including: 
  - GLP-1 medication side effect monitoring 
  - Symptom severity tracking 
  - Duration tracking 
- Structured reporting of challenges encountered between visits 

#### 9. Technical Requirements 

- Cross-platform functionality via React Native/Expo supporting iOS and Android devices 
- Responsive provider dashboard for desktop, laptop, and iPad viewing 
- Cloud-based HIPAA-compliant storage utilizing Supabase security features 
- No direct EHR integration in V1 (snapshot export system instead) 
- Manual data entry workflows optimized for efficiency 
- HIPAA compliance implementation including: 
  - Data encryption during transmission and storage 
  - Controlled access to PHI 
  - Comprehensive access logging 
  - Secure coding practices 
- Excel export capability for CDC reporting requirements 
- With V1, we're laying the groundwork for a future integration with wearable to track fitness, and future integrations with Apple & Google Fitness. These integrations with wearables & Google/Apple fitness will be added in V2.
